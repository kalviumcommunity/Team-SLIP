# Model Documentation — Loan Default Predictor

## 1. Problem Statement

Predict whether a borrower will default on their loan based on pre-decision financial
and demographic features. This is a **binary classification** problem:
- Class 0: No Default (majority — ~88.4%)
- Class 1: Default (minority — ~11.6%)

The goal is to help lenders identify high-risk loan applications before approval,
reducing potential financial losses.

## 2. Dataset

- **Source:** [nikhil1e9/loan-default](https://www.kaggle.com/datasets/nikhil1e9/loan-default) (Kaggle)
- **File:** `Loan_default.csv` (24.8 MB)
- **Rows:** 255,347
- **Columns:** 18 (1 ID, 9 numerical, 7 categorical, 1 target)
- **Class Distribution:** 225,694 No Default (88.4%) / 29,653 Default (11.6%)
- **Imbalance Ratio:** 7.6:1

## 3. Features Used

### Numerical Features (9)

| Feature | Description | Range |
|---------|-------------|-------|
| Age | Borrower's age | 18–69 |
| Income | Annual income ($) | ~15K–150K |
| LoanAmount | Requested loan amount ($) | ~5K–250K |
| CreditScore | Credit score | 300–850 |
| MonthsEmployed | Months at current employment | 0–120 |

### Categorical Features (7)

| Feature | Categories |
|---------|-----------|

| MaritalStatus | Single, Married, Divorced |

## 4. Features Excluded

| Feature | Reason |
|---------|--------|
| LoanID | Unique identifier — not a predictive feature |

**Note:** This dataset has **no leakage columns**. All features are pre-decision data
available before the loan outcome is known.

## 5. Preprocessing Steps

All preprocessing is wrapped in a `sklearn.pipeline.Pipeline` to prevent data leakage:

1. **Numerical features:**
   - `SimpleImputer(strategy='median')` — fills missing values with column median
   - `StandardScaler()` — transforms to zero mean, unit variance

2. **Categorical features:**
   - `SimpleImputer(strategy='most_frequent')` — fills missing values with mode
   - `OneHotEncoder(handle_unknown='ignore', sparse_output=False)` — binary columns per category

3. **Combined via `ColumnTransformer`** — applies appropriate transformations to each feature type

## 7. Final Model Selected

The model with the highest Test F1 on class 1 is selected as the final model.
See `docs/model_results.csv` for actual numbers after training.

**Selection criteria (priority order):**
1. Highest Test F1 on class 1
2. Recall > Precision (missing a defaulter is more costly than a false alarm)
3. Interpretability (can you explain why the model flagged someone?)

## 9. Limitations

1. **Synthetic-looking data:** The dataset features (e.g., uniform Income distribution) suggest
   it may be synthetically generated. Model performance on real lending data may differ significantly.

2. **SMOTE creates artificial samples:** Synthetic minority samples may not represent
   real-world borrower profiles, potentially inflating recall metrics.

3. **No temporal validation:** The model does not account for time-based economic changes.
   A model trained on 2020 data may not generalize to 2025 economic conditions.

4. **CreditScore may encode historical bias:** Credit scoring systems have known disparities
   across demographic groups. The model inherits these biases.

5. **Not a regulatory tool:** This model is for educational purposes. Real lending decisions
   require regulatory-compliant credit assessment methods (e.g., FICO, ECOA compliance).

6. **KNN trained on subsample:** Due to speed constraints, KNN uses only 30,000 of 255,347
   training rows. Its performance may improve with more data.

7. **No feature engineering:** We use the 16 raw features as-is. Derived features
   (e.g., LoanAmount/Income ratio) could improve performance.

## 10. Assumptions

- All features are available at loan application time (pre-decision data)
- Class distribution (~88/12) is representative of the target population
- Missing values (if any) are missing at random
- Feature distributions remain stable over time
